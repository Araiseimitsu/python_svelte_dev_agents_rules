# decisions.md — 設計判断と理由

複数の選択肢から設計・方針を選んだとき、その判断と理由を記録する。
仕様・構成の変更履歴（What）は `.docs/update.md` に書き、ここには判断の背景（Why/How）を書く。
記録したら `MEMORY.md` にインデックスを1行追記する。

## フォーマット

```markdown
## YYYY-MM-DD タイトル（何を決めたか）

- **判断**: 選んだ方針
- **選択肢**: 検討した他の案
- **理由**: なぜこれを選んだか（トレードオフ）
- **前提**: この判断が覆る条件（前提が変わったら見直す）
```
