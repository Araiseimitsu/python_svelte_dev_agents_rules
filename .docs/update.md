# 変更履歴

## 2026-07-17 レビュー指摘の適用（機械化移管ポリシー・形骸化対策・逃げ道の追加）

- **機械化移管ポリシーを新設**（AGENTS.md 記載方針）: 機械検査できるルール（依存方向・行数制限・fetch 直呼び禁止など）は実装開始後に lint / CI（`import-linter` / `eslint-plugin-boundaries` / `dependency-cruiser` 等）へ移管し、移管完了したルールは本文書から削除する。文章より機械的強制を優先する原則を明文化。
- **code-simplifier の形骸化対策**（§15）: 発動閾値を 30行 → 100行に引き上げ。非該当時の儀式的な「確認しました」報告を禁止した。
- **層構造に逃げ道を追加**（§7 依存方向、backend-python.md、frontend-svelte.md）: 空のパススルー層の作成を禁止し、小規模プロジェクトでは隣接層の統合（Backend: application + domain）や Application 層の省略（Frontend: 単純取得時のみ）を許可。依存の一方向性は維持する。
- **メモリの肥大化対策**（§17、MEMORY.md）: `MEMORY.md` インデックスに最大20行の機械的上限を設定。超過時は追記前に統合・削除する。
- **テンプレートから個人環境パスを除去**（providers.md）: `C:/Users/seizo/.gemini/...` → `~/.gemini/...` に置換し、環境非依存化した。

## 2026-07-17 ルールのスリム化（一般論の間引き・正本一元化・参照の頑健化）

- **背景**: 現行 LLM が既定で行う一般的ベストプラクティスの記載が多く、本当に重要なルール（オプトイン原則・安全ゲート）の相対的な目立ち度が下がっていた。また AGENTS.md と workflow.md の重複記載が矛盾の温床になっていた。
- **AGENTS.md に記載方針を明文化**: 「プロジェクト固有の決定事項」と「安全ゲート」のみ記載し、一般論は書かない。各ルールの正本は1ファイルに固定する。今後の追記もこの方針に従う。
- **一般論を削除**: 旧§5「プロジェクト種別の自動判定」を削除。旧§12「コード品質」・旧§15「デバッグ・ログ」を統合し、固有ルール（1,000行制限・秘密情報のログ禁止・エラー握りつぶし禁止）のみ残した新§11「コード品質・ログ」とした。§2 基本方針・workflow.md §4 デバッグ規律も同方針で圧縮し、workflow.md 旧§8「コンテキスト・効率の規律」を削除した。
- **正本の一本化**: 開発プロセス（手順・完了の定義・ループ手順）の正本を `.docs/workflow.md` に固定。AGENTS.md §3・§16 は絶対規律と参照のみに縮小し、手順の重複記載を解消した。
- **再採番と参照の頑健化**: 旧§6〜§22 → 新§5〜§20 に再採番。全ファイルの節参照を「番号＋節名」表記（例: §14 安全確認）に変更し、将来の再採番に対する耐性を上げた。ルールの実質的な内容変更はない（削除したのは一般論のみ）。

## 2026-07-15 マルチエージェント・ループ開発を完全オプトイン化（シングルエージェント既定）

- **背景**: 運用の結果、マルチエージェント・ループ開発は遅く、文脈分断により精度も低下することが判明。シングルエージェントの方が精度・効率ともに優れるとのユーザー判断。
- **AGENTS.md §18 を全面改訂**: 提案義務（30行以上・複数ファイル/レイヤー等の発動条件）を廃止。既定をシングルエージェントとし、マルチエージェント・ループ開発はユーザーの明示的な依頼時のみ実行、エージェント側からの提案・示唆を禁止した。§0 チェックリストの提案判定項目・§3 の提案義務も削除。
- **`.docs/workflow.md` §5・§6 を改訂**: §5 の委譲対象を「広範囲調査の探索委譲」「バックグラウンド実行」に限定し、実装タスクの委譲・並列分担を非推奨化。§6 の提案フォーマットを削除し、レビュー担当の既定を自己レビューに変更（外部委譲はユーザー指定時のみ）。
- **禁止事項を更新**: 「承認なしのループ開発開始」→「明示的な依頼なしのループ開発・マルチエージェント並列実装の開始・提案」に強化。
- **AGENTS.md §17 に実行主体を明記**: code-simplifier はサブエージェント委譲ではなく、実装したエージェント自身がインラインで実施する（自動実行の条件は維持）。
- **AGENTS.md §18 に優先関係を追記**: グローバルルール側の自動委譲トリガー（失敗回数による外部 AI 委譲等）より本節が優先される。
