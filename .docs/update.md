# 変更履歴

## 2026-07-15 マルチエージェント・ループ開発を完全オプトイン化（シングルエージェント既定）

- **背景**: 運用の結果、マルチエージェント・ループ開発は遅く、文脈分断により精度も低下することが判明。シングルエージェントの方が精度・効率ともに優れるとのユーザー判断。
- **AGENTS.md §18 を全面改訂**: 提案義務（30行以上・複数ファイル/レイヤー等の発動条件）を廃止。既定をシングルエージェントとし、マルチエージェント・ループ開発はユーザーの明示的な依頼時のみ実行、エージェント側からの提案・示唆を禁止した。§0 チェックリストの提案判定項目・§3 の提案義務も削除。
- **`.docs/workflow.md` §5・§6 を改訂**: §5 の委譲対象を「広範囲調査の探索委譲」「バックグラウンド実行」に限定し、実装タスクの委譲・並列分担を非推奨化。§6 の提案フォーマットを削除し、レビュー担当の既定を自己レビューに変更（外部委譲はユーザー指定時のみ）。
- **禁止事項を更新**: 「承認なしのループ開発開始」→「明示的な依頼なしのループ開発・マルチエージェント並列実装の開始・提案」に強化。
- **AGENTS.md §17 に実行主体を明記**: code-simplifier はサブエージェント委譲ではなく、実装したエージェント自身がインラインで実施する（自動実行の条件は維持）。
- **AGENTS.md §18 に優先関係を追記**: グローバルルール側の自動委譲トリガー（失敗回数による外部 AI 委譲等）より本節が優先される。

## 2026-07-07 ルールの見逃し対策（チェックリスト新設・AGENTS.md 軽量化）

- **AGENTS.md に §0「タスク開始時チェックリスト」を追加**: メモリ参照・二重実装確認・ループ提案判定・破壊的操作の承認段取り・フロントエンドルール参照・完了前検証の6項目を冒頭に集約し、長文ルールの見逃しを防ぐ。
- **AGENTS.md を参照型に軽量化**: 常時遵守ルールと参照情報を分離した。§6 のコマンド例・推奨構造は `.docs/backend-python.md` へ、§10 の env 記入例は `.docs/env-examples.md` へ、§20 の gws コマンド詳細は `.docs/providers.md` §5 へ移動（各節に「該当作業時は必ず参照」と明記）。必須・禁止ルール自体は AGENTS.md に残しており、ルールの内容変更はない。

## 2026-07-07 マルチエージェント・ループ開発とメモリ（自己成長）ルールの新設

- **AGENTS.md に §18「マルチエージェント・ループ開発」を追加**: 条件該当タスク（30行以上・複数ファイル/レイヤー・業務ロジック/API/DB・新規モジュール）では計画フェーズでループ開発を必ず提案し、ユーザー承認時のみ実行する承認ゲートを定義。章番号を再採番した（旧§18〜§20 → §20〜§22）。
- **AGENTS.md に §19「メモリ・自己成長」を追加**: 全エージェント共有のファイルベースメモリ `.docs/memory/` を新設。タスク着手時の参照義務、記録タイミング（指摘・失敗原因・環境知識・設計判断）、秘密情報の記録禁止、古いメモの淘汰ルールを定義。
- **`.docs/workflow.md` に §6「ループ開発」を追加**: 提案フォーマット、実装→別視点レビュー→修正の手順、終了条件（Critical 指摘ゼロ or 最大3周で停止・報告）を定義。基本ループの理解/報告フェーズにメモリの参照・記録を組み込んだ。章番号を再採番した（旧§6〜§8 → §7〜§9）。
- **`.docs/memory/` を新設**: MEMORY.md（インデックス）＋カテゴリ別4ファイル（pitfalls / environment / decisions / preferences）の雛形を作成。`update.md`（What の履歴）と `memory/`（How の行動改善知識）の棲み分けを明記。
- **禁止事項に追加**: 承認なしのループ開発開始、`.docs/memory/` への秘密情報記録。
- **レビュー担当の優先順位を明文化**（workflow.md §6）: ①別プロバイダー委譲（連携経路がある場合）→ ②サブエージェント委譲 → ③自己レビュー（委譲不可の場合のみ、理由を報告に明記）。

- **`.docs/workflow.md` を新設**: AI の開発プロセス（理解→計画→実装→検証→報告のループ、完了の定義、テストファースト、デバッグ規律、サブエージェント・並列化の使い分け、Git 運用、停止基準）を定義した。
- **AGENTS.md に §3「開発プロセス」を追加**: 検証なしの完了報告禁止・3回失敗で停止などの最低限の規律を本体に明記し、詳細は workflow.md へ誘導。章番号を再採番した（旧§3以降が+1）。
- **禁止事項に追加**: 検証なしの完了報告、エラー握りつぶし・テスト skip・lint 無効化による検証回避。
